## 一、产品概述

### 1.1 产品背景

Git worktree 是 Git 2.5+ 引入的功能，允许同一仓库拥有多个工作目录，非常适合并行开发场景。然而，目前市面上缺乏**免费、轻量、跨平台**的可视化管理工具：

| 工具 | 问题 |
|------|------|
| GitKraken | 付费，且功能臃肿 |
| GitLens | 依赖 VS Code |
| Git Extensions | 仅 Windows，界面老旧 |
| 命令行 | 学习成本高，无可视化 |

### 1.2 产品定位

**轻量级、免费、跨平台的 Git Worktree 可视化管理器**

- 一句话描述：让 Git worktree 像切蛋糕一样简单
- 目标用户：开发者（尤其是需要频繁切换分支的程序员）
- 核心价值：提升并行开发效率，降低 worktree 使用门槛

### 1.3 目标用户

| 用户画像 | 场景 | 痛点 |
|----------|------|------|
| **全栈开发者** | 同时开发多个功能分支 | 切换分支需要 stash，麻烦 |
| **团队 Tech Lead** | Code Review + 自己开发 | 需要同时看多个分支代码 |
| **开源贡献者** | 同时处理多个 PR | 每个 PR 一个 worktree |
| **运维/DevOps** | 紧急 hotfix | 快速切换，不影响当前工作 |

### 2.2 功能清单

## 三、用户故事

### 3.1 核心用户故事

#### US-001: 查看 worktree 列表

```
作为一个 开发者
我想要 一眼看到所有 worktree 的状态
以便于 快速了解当前并行开发的分支情况

验收标准:
- [ ] 显示所有 worktree 的分支名、路径、状态
- [ ] 状态用颜色和图标区分 (绿色=clean, 黄色=dirty, 红色=conflict)
- [ ] 显示每个 worktree 的最后提交时间和消息
- [ ] 支持按名称/状态/时间排序
- [ ] 支持关键词搜索过滤
```

#### US-002: 创建新 worktree

```
作为一个 开发者
我想要 快速创建一个新的 worktree
以便于 开始在新分支上工作而不影响当前工作

验收标准:
- [ ] 支持选择现有分支创建 worktree
- [ ] 支持输入新分支名创建 worktree
- [ ] 支持自定义 worktree 存放路径
- [ ] 自动检测路径冲突并提示
- [ ] 创建成功后自动刷新列表
```

#### US-003: 删除 worktree

```
作为一个 开发者
我想要 安全删除一个不再需要的 worktree
以便于 保持工作区整洁

验收标准:
- [ ] 删除前检查是否有未提交更改
- [ ] 有未提交更改时弹出确认对话框
- [ ] 显示即将删除的 worktree 信息
- [ ] 删除成功后自动刷新列表
- [ ] 删除失败时显示错误原因
```

#### US-004: 在 IDE 中打开 worktree

```
作为一个 开发者
我想要 一键在 IDE 中打开某个 worktree
以便于 快速开始编码

验收标准:
- [ ] 支持配置默认 IDE (VS Code / WebStorm / 等)
- [ ] 点击即可在配置的 IDE 中打开
- [ ] 首次使用时引导配置 IDE
```

### 3.2 用户旅程

```
用户打开应用
    ↓
选择/添加 Git 仓库
    ↓
查看 worktree 列表
    ↓
┌───────────────────────────────────────┐
│  [创建 worktree] → 选择分支 → 设置路径 → 完成  │
│  [删除 worktree] → 确认 → 完成              │
│  [打开 IDE]    → 在 IDE 中打开              │
│  [打开终端]    → 在终端中打开               │
└───────────────────────────────────────┘
```

## 四、非功能需求

### 4.1 性能要求

| 指标 | 要求 |
|------|------|
| 启动时间 | < 2 秒 |
| worktree 列表加载 | < 500ms (100 个 worktree 内) |
| 操作响应 | < 100ms |
| 内存占用 | < 100MB (空闲状态) |
| 安装包大小 | < 50MB |

### 4.2 兼容性要求

| 维度 | 支持 |
|------|------|
| 操作系统 | macOS 10.15+, Windows 10+, Ubuntu 18.04+ |
| Git 版本 | Git 2.5+ (建议 2.17+) |
| 架构 | x64, arm64 (Apple Silicon) |

### 4.3 可用性要求

- 界面简洁直观，新用户无需教程即可上手
- 支持键盘快捷键操作
- 支持深色/浅色主题
- 中英双语支持

### 4.4 安全性要求

- 不存储 Git 凭据
- 不上传任何用户数据
- 操作前进行安全检查
- 提供操作撤销能力（有限）

### 5.2 状态指示

| 状态 | 颜色 | 图标 | 说明 |
|------|------|------|------|
| Clean | 🟢 绿色 | ◉ | 工作区干净，无未提交更改 |
| Dirty | 🟡 黄色 | ◉ | 有未提交的更改 |
| Conflict | 🔴 红色 | ◉ | 存在冲突 |
| Detached | ⚪ 灰色 | ○ | HEAD detached 状态 |
